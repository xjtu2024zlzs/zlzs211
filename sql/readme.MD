# 数据库 SQL 提交说明

为便于系统集成和统一部署，麻烦各课题组提供本课题相关的数据库 SQL 代码，主要包括菜单栏 SQL 代码和本课题新增业务表 SQL 代码。

## 1. 菜单栏 SQL 代码

各课题如需在 RuoYi 系统中新增菜单、页面、按钮权限等内容，请提供完整的菜单栏 SQL 代码。

菜单 SQL 应包括但不限于：

```text
1. 目录菜单
2. 页面菜单
3. 路由地址
4. 组件路径
5. 菜单排序
```

请确保 SQL 可直接在 RuoYi-Cloud 当前数据库环境中执行。

---

## 2. 新增业务表 SQL 代码

各课题如需新建业务数据表，统一放在 `ry-cloud` 数据库中。

数据库名称统一为：

```sql
ry-cloud
```

各课题新建表时，表名必须添加课题前缀，避免不同课题之间表名冲突。

表名前缀规则如下：

| 课题  | 表名前缀  | 示例       |
| --- | ----- | -------- |
| 课题一 | `t1_` | `t1_xxx` |
| 课题二 | `t2_` | `t2_xxx` |
| 课题三 | `t3_` | `t3_xxx` |
| 课题四 | `t4_` | `t4_xxx` |
| 课题五 | `t5_` | `t5_xxx` |

例如课题五新增质量反馈表，表名应写为：

```sql
t5_quality_feedback
```

不建议使用如下无前缀表名：

```sql
quality_feedback
feedback_record
algorithm_result
```

---

## 3. SQL 提交要求

各课题提交 SQL 时，建议按以下结构整理：

```text
sql
├── menu.sql          # 菜单栏 SQL
├── table.sql         # 新增业务表 SQL
└── init_data.sql     # 初始化数据 SQL，如无可不提供
```
