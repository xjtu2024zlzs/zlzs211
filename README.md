# 项目整体架构与模块说明

## 1. 项目概述

本项目是一个基于 RuoYi-Cloud 微服务框架的综合性平台，主要包含以下几个核心部分：

1. **RuoYi-Cloud 微服务框架** - 基础微服务架构（不包含在本说明中）
2. **算法服务模块** - Python/FastAPI 算法服务
3. **多课题业务模块** - 5个独立的课题研究模块
4. **配置管理** - Nacos 配置中心
5. **数据库设计** - MySQL 数据库结构



## 2. Python 算法服务规范

### 2.1端口分配

为避免端口冲突，各课题 FastAPI 服务端口统一分配：

| 课题  | 服务目录              | FastAPI 端口 |
| --- | ----------------- | ---------: |
| 课题一 | `python/project1` |       9701 |
| 课题二 | `python/project2` |       9721 |
| 课题三 | `python/project3` |       9741 |
| 课题四 | `python/project4` |       9761 |
| 课题五 | `python/project5` |       9781 |
详见 `python/` 目录下的 `README.md` 文件。


### 2.2 开发约定
1不建议将 Python 算法代码放入 RuoYi 模块中
2. 各课题必须提供 `requirements.txt`
3. 各课题应提供基本 `README.md` 说明
4. 不得使用默认端口，必须使用指定端口


## 3. 多课题业务模块

### 3.1 课题分布

项目包含5个独立的课题研究模块，分别对应不同的业务场景：

- **课题一**: 位于 `ruoyi-modules/ruoyi-project1`，主要涉及档案管理系统
- **课题二**: 待实现 
- **课题三**: 位于 `ruoyi-modules/project3`
- **课题四**: 待实现
- **课题五**: 位于 `ruoyi-modules/ruoyi-topic5`

### 3.2 数据库表命名规范

为避免不同课题间表名冲突，采用前缀区分：

| 课题  | 表名前缀  | 示例       |
| --- | ----- | -------- |
| 课题一 | `t1_` | `t1_xxx` |
| 课题二 | `t2_` | `t2_xxx` |
| 课题三 | `t3_` | `t3_xxx` |
| 课题四 | `t4_` | `t4_xxx` |
| 课题五 | `t5_` | `t5_xxx` |
详见 `sql/` 目录下 `README.md` 文件。

## 4. 配置管理

### 4.1 Nacos 配置

各课题配置文件位于 `nacos/` 目录

详见 `nacos/` 目录下的 `README.md` 文件。

### 4.2 配置文件命名规范

配置内容包括：
- 数据库连接
- Redis 配置
- 服务端口
- 服务名称
- 第三方接口地址
- FastAPI 调用地址

## 5. 数据库设计

- 统一数据库名称: `ry-cloud`
- 各课题表必须添加前缀(`t1_`, `t2_`, `t3_`, `t4_`, `t5_`)
- 菜单SQL需包含目录、页面、路由、组件路径、排序等信息
- 建议按菜单、表结构、初始化数据分类整理


## 6. 开发流程

### 新增课题开发步骤

1. **创建目录结构**
   - 在 `ruoyi-modules/` 下创建 `ruoyi-projectX`
   - 在 `python/` 下创建 `projectX`
   - 在 `nacos/` 下创建 `projectX` 配置目录
   - 在 `sql/` 下创建 `projectX` SQL目录

2. **配置端口**
   - Java 服务端口: 参照 RuoYi 规范
   - Python FastAPI 端口: 按分配规则设置

3. **数据库设计**
   - 创建带前缀的业务表
   - 编写菜单权限SQL
   - 准备初始化数据

4. **配置文件**
   - 创建 Nacos 配置文件
   - 配置数据库连接
   - 配置服务发现

5. **算法服务**
   - 实现 FastAPI 接口
   - 编写 `requirements.txt`
   - 创建课题 README
